## 강의 목표

이 온보딩은 Claude Code 사용법을 기능 단위로 설명하는 교육이 아니라, Brand Genome의 업무 기준을 AI 작업 환경에 이식해 조직 자산으로 축적하는 방식을 이해시키는 교육입니다.

직원에게 전달할 핵심 문장은 다음과 같습니다.

> 앞으로 AI를 각자 감으로 쓰는 것이 아니라, 회사 기준이 들어 있는 하네스 폴더 안에서 같은 방식으로 쓰게 됩니다.

교육 후 직원이 이해해야 할 내용은 세 가지입니다.

| 목표 | 설명 |
| --- | --- |
| Claude Code의 위치 이해 | 일반 챗봇이 아니라 회사 폴더의 규칙, 예시, 템플릿을 읽고 결과물을 만드는 작업 환경으로 이해한다 |
| 하네스의 역할 이해 | 하네스 폴더를 회사의 보고서 양식, 카피 규칙, 상품기획 기준, 마케팅 분석 방식, SCM 체크리스트가 쌓이는 기준서로 이해한다 |
| 실무 적용 흐름 이해 | 팀별 업무 재료를 입력하고, 결과물을 검토하고, 좋은 사용법을 AX 담당자에게 전달하는 운영 흐름을 이해한다 |

## 세션별 진행안

### 1. 강의 개요: 왜 Claude Code를 도입하는가

일반 Claude, GPT, Gemini는 대화창에서 답변을 받는 도구에 가깝습니다. Claude Code는 회사 폴더 안의 규칙, 예시, 템플릿, 스킬을 읽고 그 기준에 맞춰 업무 결과물을 만드는 작업 환경입니다.

이 세션에서는 직원들이 Claude Code를 개발자 전용 도구로 오해하지 않도록, “AI가 회사 업무 매뉴얼을 읽고 도와주는 공간”이라는 설명을 먼저 제공합니다.

### 2. Claude Code와 하네스 이해

하네스 폴더는 Brand Genome의 업무 기준이 들어 있는 AI 운영 폴더입니다. 보고서 양식, 카피 규칙, 상품기획 기준, 마케팅 분석 방식, SCM 체크리스트, 글로벌 현지화 기준이 이 안에 쌓입니다.

직원에게 보이는 기본 파일은 다음 네 가지로 잡습니다.

| 파일 | 역할 |
| --- | --- |
| `00_START_HERE.md` | 처음 시작하는 안내문 |
| `01_WORK_INPUT_CARD.md` | 업무 재료 입력 카드 |
| `02_SAMPLE_OUTPUTS.md` | 좋은 산출물 예시 |
| `03_SUBMIT_GUIDE.md` | 저장·제출·공유 방식 |

### 3. 역할 분담

| 역할 | 해야 할 일 |
| --- | --- |
| 경영진 / C-Level | AI가 따라야 할 브랜드 방향성, 제품 철학, 최종 판단 기준 승인 |
| AX 담당자 | 하네스 폴더, 스킬, 템플릿, GitHub 저장소 관리 |
| 팀 리더 | 팀별 업무 기준 검수, 스킬 후보 승인·수정·반려 |
| 실무자 | Claude Code로 실제 업무 수행, 좋은 입력 사례·실패 사례·반복 절차 제보 |

직원은 하네스를 고치는 사람이 아니라 하네스를 활용해 업무를 수행하는 사람입니다. 좋은 사용법이 발견되면 AX 담당자에게 전달하고, AX 담당자가 이를 공식 스킬, 템플릿, 체크리스트로 승격합니다.

### 4. 직원 사용 원칙

| 구분 | 내용 |
| --- | --- |
| 해야 할 일 | 업무 재료를 입력 카드에 맞춰 정리하고, Claude Code 결과물을 검토한 뒤 제출한다 |
| 제보할 일 | 좋은 입력 사례, 실패 사례, 반복되는 절차, 팀별로 자주 쓰는 입력값을 AX 담당자에게 전달한다 |
| 하면 안 되는 일 | 하네스 폴더 구조를 임의로 바꾸거나, 스킬 파일과 템플릿 파일을 직접 수정하거나, AI 결과물을 검토 없이 제출하지 않는다 |

## 팀별 실습 예시

팀별 실습은 확정된 업무 배정표가 아니라, 교육 중 각 팀이 자기 업무와 연결해서 이해하도록 돕는 예시입니다.

| 팀 | 실습 과제 | 입력 재료 | 결과물 |
| --- | --- | --- | --- |
| 상품기획팀 | 신제품 콘셉트 초안 만들기 | 제품군, 타깃 고객, 핵심 효능, 경쟁 제품, 가격대, 차별점 | 제품 콘셉트, USP, 상세페이지 구조, 디자인 브리프 초안 |
| 마케팅팀 | 캠페인 콘텐츠 기획안 만들기 | 제품명, 캠페인 목적, 타깃, 채널, 레퍼런스, 금지 표현 | 광고 카피, 인플루언서 브리프, 콘텐츠 후킹 문구, 성과 회고 틀 |
| 그로스팀 | 리뷰·VOC에서 개선 포인트 찾기 | 리뷰, CS 문의, 불만 키워드, 채널별 매출 지표 | VOC 요약, 반복 불만 패턴, FAQ 개선안, 상세페이지 보완점 |
| SCM / 글로벌 TF | 런칭 준비 체크리스트 만들기 | 발주 일정, 입고 일정, 재고 현황, 국가별 문구, 현지 규정 이슈 | 런칭 리스크, 체크리스트, 공급사 커뮤니케이션 초안, 미국·일본 현지화 문구 |

## HTML 보고 실습

HTML 보고는 Claude Code 교육의 전체 목적이 아니라 첫 번째 공통 활용 사례입니다. 보고는 모든 팀이 공통으로 수행하는 업무이므로, `/report`를 첫 공식 실습으로 사용하면 전사 공통 사용법을 빠르게 익힐 수 있습니다.

보고 유형은 두 가지로 설명합니다.

| 유형 | 기준 | 최종 산출물 |
| --- | --- | --- |
| 의사결정 필요 보고 | 승인, 반려, 선택, 보류, 우선순위 조정이 필요할 때 | 선택지 2~3개와 작성자 추천안 |
| 현황 공유 보고 | 결정 없이 진행 상황, 이슈, 다음 액션을 공유할 때 | KPI, 표, 그래프, 캡처 중심 HTML |

최종 HTML에는 작성 중 오간 내부 대화나 작업 절차가 드러나지 않아야 합니다. 화면에는 완성된 보고서만 남깁니다.

## 운영 메모

| 구성 원칙 | 이유 |
| --- | --- |
| 이론을 먼저 설명 | 직원들이 Claude Code를 개발자용 툴로 오해하지 않도록, 회사 업무 기준을 AI와 함께 쓰는 환경이라는 점을 먼저 이해시킨다 |
| 팀별 실습은 중간에 배치 | 도구 설명만 하면 각 팀이 자기 업무와 연결하기 어렵기 때문에, 팀별 예시 과제를 통해 바로 적용해본다 |
| HTML 보고는 마지막에 배치 | 처음부터 HTML을 강조하면 보고서 제작 도구로만 이해할 수 있으므로, Claude Code의 운영 목적을 먼저 설명한 뒤 공통 활용 사례로 보여준다 |
| 좋은 사용법을 회수 | 개인의 사용 팁으로 끝내지 않고, 공식 스킬·템플릿·체크리스트로 승격해 조직 자산으로 축적한다 |
