## 핵심 요약

Brand Genome의 Claude Code 온보딩 교육안은 “코딩 도구 교육”이 아니라 회사 업무 방식과 기준을 AI 작업 환경에 이식하는 운영 교육으로 설계되었습니다.

핵심 메시지는 직원들이 AI를 각자 감으로 쓰는 것이 아니라, 회사 기준이 들어 있는 하네스 폴더 안에서 같은 방식으로 업무 결과물을 만들게 하는 것입니다.

교육 흐름은 이론 설명, 팀별 실습, HTML 보고 실습 순서가 적절합니다. 먼저 Claude Code와 하네스의 의미를 이해시킨 뒤, 상품기획·마케팅·그로스·SCM·글로벌 업무에 바로 연결하는 방식입니다.

## 주요 변화·이슈

- 직원들이 Claude Code를 개발자 전용 도구로 오해하지 않도록, “회사 업무 매뉴얼을 읽고 결과물을 만드는 작업 환경”이라는 설명이 먼저 필요합니다.
- 하네스 폴더는 회사의 보고서 양식, 카피 규칙, 상품기획 기준, 마케팅 분석 방식, SCM 체크리스트, 글로벌 현지화 기준을 축적하는 기준서로 설명해야 합니다.
- 직원은 하네스를 임의로 수정하는 역할이 아니라, 하네스를 활용해 업무를 수행하고 좋은 사례와 실패 사례를 AX 담당자에게 전달하는 역할로 정리됩니다.
- 팀별 실습은 상품기획, 마케팅, 그로스, SCM·글로벌 TF별로 실제 업무 재료를 넣고 결과물을 확인하는 방식이 적합합니다.
- HTML 보고는 첫 번째 공통 활용 사례로 다루되, 교육 전체가 보고서 작성 도구 교육으로만 보이지 않도록 마지막 단계에 배치하는 편이 좋습니다.
